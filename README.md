# AI Support API

REST API для обработки формы обратной связи с использованием AI для анализа обращений.

---

# Возможности

- REST API для формы обратной связи
- Валидация входных данных
- Анализ сообщений с помощью GroqAI (OpenAI-compatible)
- Определение тональности сообщения
- Классификация обращения
- Автоматическая генерация ответа пользователю
- Отправка email владельцу сайта
- Отправка копии пользователю
- Graceful fallback при недоступности AI
- Rate Limiting
- Логирование запросов
- Swagger/OpenAPI документация
- Метрики обращений

---

# Запуск проекта

## Требования

- PHP 8.2+
- Composer
- MySQL
- [Groq API Key](https://console.groq.com/keys)

## Установка

### Требования

* Docker
* Docker Compose

### Запуск

Клонировать репозиторий:

```bash
git clone <repository>
cd project
```

Создать файл окружения:

```bash
cp .env.example .env
```

Заполнить необходимые переменные окружения:

#### Groq API

```env
OPENAI_API_KEY=your_groq_api_key
OPENAI_API_BASE_URL=https://api.groq.com/openai/v1
OPENAI_API_MODEL=openai/gpt-oss-20b
```

#### SMTP

```env
MAIL_MAILER=smtp
MAIL_HOST=your_smtp_host
MAIL_PORT=your_smtp_port
MAIL_USERNAME=your_email
MAIL_PASSWORD=your_password
MAIL_FROM_ADDRESS=your_email
MAIL_FROM_NAME="AI Support API"

ADMIN_EMAIL=admin@example.com
```

При необходимости также настройте параметры подключения к базе данных (`DB_*`).

Собрать и запустить контейнеры:

```bash
docker compose up -d --build
```

Выполнить миграции:

```bash
docker compose exec app php artisan migrate
```

Если ключ приложения отсутствует, сгенерировать его:

```bash
docker compose exec app php artisan key:generate
```

После запуска API будет доступно по адресу:

```
http://localhost:8000
```

Swagger-документация:

```
http://localhost:8000/api/documentation
```

Для остановки контейнеров:

```bash
docker compose down
```

# Архитектура

Использована слоистая архитектура.

```
Controllers
    ↓
Services
    ↓
Repositories
```

### Controllers

Принимают HTTP-запросы, валидируют данные и вызывают бизнес-логику.

### Services

Содержат бизнес-логику приложения:

- работа с AI;
- отправка писем;
- обработка обращений.

### Repositories

Работают только с базой данных.

Такое разделение упрощает поддержку и тестирование проекта.

# API

## POST /api/contact

Создание обращения.

### Пример запроса

```json
{
    "name":"Max",
    "email":"max@test.com",
    "phone":"+79999999999",
    "comment":"Не работает форма"
}
```

Ответ

```json
{
    "name": "Max",
    "email": "max@test.com",
    "phone": "+79999999999",
    "comment": "Не работает форма",
    "sentiment": "negative",
    "category": "support",
    "ai_reply": "Спасибо за обращение! Приносим извинения за неудобства. Пожалуйста, уточните, какая именно форма не работает и на каком этапе возникла ошибка, чтобы мы могли быстро помочь.",
    "updated_at": "2026-07-17T18:09:36.000000Z",
    "created_at": "2026-07-17T18:09:36.000000Z",
    "id": 31
}
```

# Валидация

Проверяются:

- имя
- email
- телефон
- комментарий

При ошибке возвращается

```
422 Validation Error
```

---

# Обработка ошибок

Используются HTTP статусы:

| Код | Описание |
|-----|----------|
|201|Создано|
|422|Ошибка валидации|
|429|Превышен лимит запросов|
|500|Внутренняя ошибка сервера|

Ошибки логируются в файл Laravel.

# AI-интеграция

Использован Groq Chat Completions API.

AI выполняет:

- анализ тональности;
- определение категории обращения;
- генерацию автоматического ответа пользователю.

Используемый промпт задает строгий JSON-формат ответа и ограничивает возможные значения полей.

---

# Graceful Fallback

Если AI недоступен:

- обращение всё равно сохраняется в базе;
- письма продолжают отправляться;
- используются значения по умолчанию:

```json
{
    "sentiment":"neutral",
    "category":"other",
    "reply":"Спасибо за ваше обращение. Мы скоро свяжемся с вами."
}
```

Работа сервиса не прерывается.

# Хранение данных

## Логи

Используется стандартная система логирования Laravel.

Файл:

```
storage/logs/laravel.log
```

Логируются:

- входящие запросы;
- ошибки AI;
- ошибки отправки email.

## Rate Limiting

Используется встроенный Laravel Rate Limiter.

Ограничение:

```
10 запросов в минуту с одного IP.
```

## Статистика

Статистика обращений хранится в базе данных и вычисляется SQL-агрегацией по сохраненным обращениям.
